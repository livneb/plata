Dead-letter queues

What is a "Dead Letter"?

When an agent receives a message from a Redis stream and tries to process it, if processing raises an exception we acknowledge the message anyway (so it doesn't block the stream forever) but copy it to a per-agent dlq:<agent> stream. After 3+ failed attempts on the same message, it lands here. Replay re-publishes it to the original source stream — use after a bug fix. Discard deletes it. An empty DLQ list means no message has crashed lately — that's the healthy state.
